

# **智能留学费用估算工具 - 产品需求文档**

**文档版本:** 1.0
**最后更新日期:** 2024-01-20
**项目阶段:** MVP

### **1. 产品愿景与目标**

**愿景:** 为申请海外本科及硕士的留学生提供最可信、最个性化的留学费用透明度工具，消除信息不对称，助力其财务规划。

**MVP核心目标:**
1.  **验证核心价值:** 证明通过LLM驱动的精准数据获取与个性化估算，能提供远超市场平均水平的可信费用报告。
2.  **收集用户反馈:** 通过最小化可行产品，快速收集用户对数据准确性、报告实用性及用户体验的反馈。
3.  **技术路径验证:** 测试以LLM为核心、结合实时联网搜索的技术架构的可行性与成本结构。

**成功指标:**
-   在前20所美/澳目标大学中，免费版报告生成成功率 > 95%。
-   通过用户反馈问卷，90%的用户认为数据“基本准确”或“非常准确”。
-   单次报告生成的平均成本控制在 `$X.XX` 以内。

### **2. 用户画像与用户故事**

**核心用户:** 计划赴美国或澳大利亚攻读本科或硕士学位的国际留学生。

**用户故事:**
-   **作为一名准留学生，** 我希望输入我的目标院校和专业，以便获得该项目的准确学费信息。
-   **作为一名对生活成本不确定的学生，** 我希望选择我的消费档次和住宿偏好，以便系统能为我生成一个贴合我生活习惯的个性化生活费估算。
-   **作为一名注重预算的学生，** 我希望在报告中看到所有主要费用的清晰分类和估算区间，以便我能全面了解总花费并制定预算。
-   **作为一名谨慎的申请者，** 我希望报告中的数据都有明确的可信来源链接，以便我能亲自核实信息的准确性。

### **3. 功能需求**

#### **3.1 核心功能流**
用户通过单一表单提交信息，系统后端并行处理多项任务，最终返回一份整合的可视化报告。

#### **3.2 功能详述**

| 模块 | 功能描述 | 优先级 | 备注与规则 |
| :--- | :--- | :--- | :--- |
| **用户输入** | 提供信息收集表单 | **必须** | **必填:** 国家、大学、项目、城市、阶段、消费档次、住宿偏好。<br>**选填:** 饮食习惯、交通偏好。 |
| **学费查询** | 实时查询并返回指定项目的国际生学费。 | **必须** | **数据源:** 大学官方网站。<br>**降级策略:** 若查询失败，返回该大学同阶段学生的平均学费，并明确告知用户此为“估算值”。 |
| **生活成本分析** | 根据用户画像推算个性化生活费。 | **必须** | **数据源:** 官方数据/Numbeo等平台的城市平均数据。<br>**计算规则:** <br> - **经济型:** 平均值 **-20%** <br> - **标准型:** 平均值 **±0%** <br> - **舒适型:** 平均值 **+25%** <br> **输出:** 提供各项费用的估算区间。 |
| **其他费用估算**| 估算申请费、签证费。 | **必须** | **数据源:** 学校官网、大使馆网站。<br>**初始安家费:** MVP阶段暂不估算。 |
| **报告生成** | 汇总所有数据，生成可视化报告。 | **必须** | **内容:** <br> 1. 总费用及分类费用饼图。<br> 2. 年度/月度视图。<br> 3. 所有数据均需注明来源链接。<br> 4. 基于用户选择的个性化省钱建议。<br> **输出格式:** 网页交互式数据面板 + 可下载的PDF报告。 |

### **4. 非功能需求**

-   **性能:** 在MVP阶段，由于采用实时查询，目标为80%的报告在 **30秒内** 生成完毕。
-   **可靠性:** 报告生成成功率 > 95%。对于任何降级数据，必须在UI上有明确标识。
-   **数据透明度:** 所有核心数据（学费、租金等）必须提供其来源链接。对于估算值，需清晰标注“估算”字样。
-   **成本:** 监控单次报告生成的API调用成本，为商业化定价提供依据。

### **5. 技术架构与数据策略**

#### **5.1 系统架构图**
系统交互流程如下所示：

```mermaid
sequenceDiagram
    actor User as 潜在留学生
    participant F as 前端界面<br>（信息表格）
    participant O as 智能估算协调器
    participant A as 学费查询助手
    participant B as 生活成本分析师
    participant C as 其他费用专员
    participant D as 报告撰写员

    User->>+F: 1. 填写个人信息与留学目标
    F->>+O: 2. 提交信息
    O->>A: 3. 查询项目学费
    O->>B: 3. 分析生活成本
    O->>C: 3. 估算其他费用
    Note over A,B,C: 并行处理任务<br>（联网搜索 & 智能推理）
    A-->>O: 返回学费数据及来源
    B-->>O: 返回个性化生活费估算
    C-->>O: 返回其他费用估算
    O->>+D: 4. 汇总所有数据
    D->>D: 生成可视化图表与个性化建议
    D-->>-F: 5. 返回完整报告
    F-->>-User: 6. 展示交互式数据面板与PDF报告
```

#### **5.2 技术选型**
-   **LLM服务:** 主用 **OpenRouter 提供的 gpt-4o API**。
-   **前端:** 网页应用。
-   **后端:** 待定（需能处理并行API调用）。
-   **数据:** MVP阶段以实时查询为主，为未来建立本地缓存数据库预留架构空间。

#### **5.3 数据策略**
-   **MVP阶段:** **全面实时查询**，以验证技术路径和数据的可获取性。
-   **未来演进:** 过渡至 **“本地缓存数据库 + 付费实时更新”** 的混合模式，以优化成本和速度。
-   **降级方案:** 明确且透明的降级策略，确保服务可用性。

### **6. 商业化考虑**

-   **MVP阶段:** 完全免费，以收集用户反馈和验证市场为主。
-   **未来路径:** 采用服务分级模式。
    -   **免费版:** 使用缓存数据 + 标准模型，生成速度快的基准报告。
    -   **付费版:** 使用实时联网数据 + 顶级模型（如GPT-4），生成最精准、最新的深度报告。
